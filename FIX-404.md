# Fix: HTTP 404 at http://localhost:8080/oceanview/

The 404 means Tomcat is running but the **oceanview** app is not deployed or failed to start.

## 1. Deploy the WAR

Run these in Terminal (use your real Tomcat path if different):

```bash
# Build the WAR
cd /Users/hamshi/Documents/OcenView
mvn clean package -DskipTests

# Copy the WAR into Tomcat's webapps folder
cp target/oceanview.war /opt/homebrew/opt/tomcat@9/libexec/webapps/
```

**If you don’t use Homebrew Tomcat**, find your Tomcat’s `webapps` folder and copy there, e.g.:

- **macOS (Homebrew):** `/opt/homebrew/opt/tomcat@9/libexec/webapps/`
- **Windows:** `C:\Program Files\Apache Software Foundation\Tomcat 9\webapps\`
- **IDE:** Use the server’s “Deploy” or “Add deployment” and choose `target/oceanview.war`, or copy the WAR into the server’s `webapps` directory shown in the IDE.

## 2. Wait for Tomcat to unpack

After copying the WAR, wait **15–20 seconds**. Tomcat unpacks `oceanview.war` into a folder `oceanview/`. Then try:

- http://localhost:8080/oceanview/

## 3. If you still get 404

- **Confirm the WAR is in webapps:**  
  List the folder:  
  `ls /opt/homebrew/opt/tomcat@9/libexec/webapps/`  
  You should see `oceanview.war` and, after a short time, an `oceanview` directory.

- **Check Tomcat logs** for deployment errors:  
  - Homebrew: `tail -100 /opt/homebrew/opt/tomcat@9/libexec/logs/catalina.out`  
  - Or in your IDE: Server → Logs / Console.  
  Look for lines containing `oceanview` or `FAILED` / `Exception`.

- **Confirm Tomcat is the one you’re using:**  
  If you start Tomcat from an IDE, the app must be deployed to that server’s `webapps`, not only to Homebrew’s.

## 4. One-command deploy (Homebrew Tomcat)

From project root:

```bash
./deploy.sh
```

This builds, stops Tomcat, removes the old app, copies the new WAR, and starts Tomcat. Wait 15 seconds, then open http://localhost:8080/oceanview/.
